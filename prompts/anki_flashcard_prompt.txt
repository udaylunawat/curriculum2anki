GOAL:
Create a minimal set of flashcards that build a Mental Model.
Optimize for distinctness, connections, and recall, not coverage.

1. Scan input for distinct concepts (entities).

   - If a concept is referenced but not explained in depth, assume it is already known.
   - Do NOT generate a definition card for prerequisites.
4. Generate cards using the Card Matrix below.

----------------------------------------------------------------
THE CARD MATRIX
----------------------------------------------------------------

1. THE ANCHOR (Definition)
Front:
What is [Concept]?

Back:
A precise, boundary-setting definition.
Focus on what makes it unique.

Rules:
- ONE anchor per concept.
- No synonyms.
- Skip if the text says "recall", "as we know", or similar.

Tag: Foundations

------------------------------------------------------------

2. THE MECHANISM (Process / Math)
Front:
How does [Concept] work?
OR
What is the objective function of [Concept]?

Back:
Explain the algorithm, geometry, or core equation.
Explain variables, not just formulas.

Rules:
- Use LaTeX for ALL math.
- Prefer explanation over raw equations.

Tag: Mechanism

------------------------------------------------------------

3. THE CONNECTION (Lateral Thinking)
Front:
How does [Concept A] relate to [Concept B]?
OR
Difference between [A] and [B]?

Back:
Explain the explicit relationship stated in the text:
trade-off, approximation, equivalence, or contrast.

Rules:
- Generate ONLY if the input explicitly links them.
- Do NOT invent comparisons.

Tag: Connections

----------------------------------------------------------------
CONTENT RULES (STRICT)
----------------------------------------------------------------
FRONT:
- Max 15 words.
- No pronouns.
- Explicitly name the concept.

BACK:
1. Technical explanation (concise).
2. <br><br><b>ELI5:</b> Intuition or analogy.
3. <i>Context: [Parent Topic]</i> only if needed.

MATH:
- Inline: $x_i$
- Block:
  $$\sum_{i=1}^{n}$$
- NEVER use HTML for math.

----------------------------------------------------------------
QUALITY CONTROL
----------------------------------------------------------------
- Zero duplicate definitions.
- Prefer fewer, stronger cards.
- If unsure, DO NOT generate a card.
